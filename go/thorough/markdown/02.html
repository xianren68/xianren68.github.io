编译过程 ​

1. 词法分析 ​

将源代码解析为token,token即是代码中不可再分的最小的语义单位

2. 句法分析 ​

将token解析为抽象语法树(AST)

3. 语义分析 ​

  1. 类型检查
  2. 类型推断
  3. 查看类型是否匹配
  4. 函数调用内联优化
  5. 逃逸分析

4. 中间码生成 ​

为处理不同平台差异生成的中间代码(SSA),是与平台无关的汇编

查看中间码的生成过程 ​

yaml
$env:GOSSAFUNC="main" // 设置环境变量(powershell为例),指定要查看的函数名

然后go build编译，可以看到生成的ssa.html文件,它就是生成中间码的过程

5. 机器码生成 ​

  1. 生成plan9汇编
  2. 将汇编转化成机器码,.a文件

查看生成的plan9汇编 ​

yaml
go build -gcflags -S

6. 链接 ​

将所有的.a文件链接成一个可执行文件
